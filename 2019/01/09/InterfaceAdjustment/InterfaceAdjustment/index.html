调整方案

版本一

  • UML类图

  • 类设计说明

    CBsReconize：提供业务需求中的所有接口

    rec_condition:识别条件参数，利用结构体指针CON_PARAM，为了识别条件扩展性，新增条件参数仅需继承CON_PARAM声明新的结构体。

    pResData:为输入输出参数，调用者根据需要识别的对象，输入对应数据结构，识别完成输出数据结构

  • 其他接口:

    LoadDwg(),加载dwg图纸数据库接口,此接口主要为了待分析的数据库数据复用，一次性加载，多次进行识别，直到更换图纸数据库dwg或生命周期结束，其中的内存才释放。未提供手动释放接口。

    SetWorkDir,主要用于设置识别所需配置文件目录

    WriteToXML,主要用于将识别结果输出到xml文件，针对北京评审。

    CON_PARAM：条件参数接口类，用于提供识别条件，其中抽取其公共属性范围作为成员属性，提供纯虚函数为接口函数，便于识别模块转换为其具体类。设计此类目的因为识别条件的不确定性，方便今后参数拓展。

    IBsRecognize：识别接口类,抽象其识别方法，具体识别方法位于其实现类中，对外隐藏了其内部实现。方便外部接口调用。调用者只需关心自己需要识别什么，无需关心具体利用哪个类去进行识别工作。

  • 其他类

    TAG_FRAME_DATA、TAG_DIRTEXT_DATA...等，主要为识别结果数据结构，用作输出，供其他C++调用者使用

版本二

  • UML类图

  • 上一版本问题说明

    • 接口类CBsReconize提供的识别接口中的输出参数为具体数据结构，而识别类IBsRecognize中提供抽象识别方法无法进行数据结构传入，导致接口类与识别类无法正常交互。陷入瓶颈。
    • 识别类的具体类的实例化，应该由谁进行实例化？谁负责释放？按上一版本设计，只能在各自的接口识别方法中进行各自实例化操作。若出现业务拓展需要新增接口，则调整太大。
    • 其输出数据类型为一个聚合类，平台方调用需要实例化对应数据结构，才能进行XML写入，增加类调用方使用复杂度。
    • XML该由谁负责写入？

  • 问题解决

    • 新增枚举类型REC_TYPE，将接口类中的接口调整为一个，根据传入枚举值进行是识别。
    • 新增工厂类，进行识别类的生产实例化工作，公接口类调用
    • 新建TAG_BASE_DATA基类，输出数据结构进行抽象，作为识别接口输入输出参数，接口类需要精确输入其数据结构具体类实例指针，否则可能导致崩溃或数据紊乱
    • 将XML写入工作交给具体识别类去负责

版本三

  • UML类图 ‪

  • 上一版本问题说明

    • 对接口Recognizein第三个数据参数要求过高，需要调用者精确输入待识别数据结构实例化对象，否则导致不可预料的错误发生。
    • XML写入方法放置在具体识别类中，当对应数据类发生改变时，维护代码者容易疏忽更新对应的XML写入方法

  • 问题解决

    • 传入数据结构基类二级指针，由具体识别类去实例化具体数据结构。因为具体识别类了解其具体数据结构。
    • XML交给具体数据类去写入，作为数据结构本身去处理较为合适，避免了当数据结构变更时，代码维护者忽略对XML写入的同步更新。

  • 新增

    • 预处理类
    • 配置类
    • 预处理图纸基础数据类

最终结果

  • UML类图

  • 上一版本问题说明

    • 类名容易引起误会，CBsRecognize 与 IBsRecognize
    • 预处理数据该由哪个类进行管理？
    • 识别模块部分具体类可进行再次抽象
    • 当系统存在多个配置时，无法切换工作目录

  • 问题解决

    • 将类CBsRecognize更名为CBsRecManager，更贴近其类职责
    • 预处理应该只负责数据处理，不负责数据管理
    • 现将目录表识别，材料表识别，图框识别的基类设为通用表格识别
    • 恢复版本一中设置工作目录方法

  • 新增

    • 日志记录类

  • 其他问题

    因提供平台使用错误码信息的头文件不大合适，可提供接口函数GetErrorString(err_code);返回具体错误信息 同时可将错误信息写入日志，平台可跟踪日志了解错误信息

/**
 *加载dwg
 */
LoadDwg(dwg图纸路径)
{
	// 其中包括预处理操作
	// 预处理读取的所有所有数据保存在
	// CBsRecManager类中属性m_pBlockTable
	
	// 其中需要判断如果是同一个数据库已经加载则不再重复加载
	// 或加载前清理所有常驻内存
	// 若预处理比较耗时，可考虑将已经处理过的数据序列化
	CBsRecManager::instance().LoadDwg(dwg图纸路径);
}


/**
 *卸载dwg预处理及其他常驻数据
 */
UnLoadDwg()
{
	// 清理
	// CBsRecManager::instance().UnLoadDwg();
}

/**
 *主要用于识别,输出内存数据
 */
RecognizeDwg(type,con_param*,outdata**)	
{
	// 开始识别
	CBsRecManager::instance().Recognize(type,con_param*,outdata**);
}

/**
 *主要用于识别，输出xml
 */
RecognizeDwg(type,con_param*, lpszXml)
{
	// 数据转换，适配另一个重载识别方法
	RecognizeDwg(type,con_param*,outdata**);

	// 识别后的数据写入xml
	outdata-toXml();
}

/**
 *设置工作目录
 */
SetWorkDir(lpszPath)
{
	CBsRecManager::instance().SetWorkDir(lpszPath);
}
