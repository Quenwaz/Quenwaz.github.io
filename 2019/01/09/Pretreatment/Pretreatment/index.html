功能概括

针对单个大文件预处理时间较长，且需要多次使用。所以单独提取预处理模块

现状概括

1. 图例识别

  • 读取数据库中所有实体，按块表记录存储。
  • 标准图例特征提取
  • 待识别文件特征提取
  • 待识别文件部分块及多段线进行分解，统计各块表记录文字大小
  • 图元分析及扫描器构建（用于快速检索）

    2. 通用表格识别

  • 按块读取数据库中所有实体与块,按块进行表格识别，根据矩阵复制

    3. 封面识别

  • 依赖图例识别提供的方法，获取数据库中所有块实体
  • 读取块列表中所有实体中的文本

    4. 材料表识别

  • 主要依赖通用表格识别

    5. 目录表识别

  • 主要依赖通用表格识别

    可调整项

    一、 公共基础数据结构调整

  • 现有结构来由

    现有基础数据结构分别由BsEntity,及GaEntity构建，两者可相互转换。之所有存在两个基本数据结构类，由于不同阶段开发人员不同，所以根据不同阶段的需求，不同开发人员分别构建了这两个基础数据结构

  • 现有结构分析

    1. 基础数据结构类BsEntity与GaEntity构建出发点一致，都为基础数据结构封装类。
    2. 基础数据结构中BsEntity及GaEntity类功能复杂，负责多种类型实体的管理。

  • 调整方向

    1. 将BsEntity及GaEntity，构建成一个基础数据结构
    2. 构建一个基础类，多种类型实体继承此基础类，进行不同的实现处理

二、 基础预处理调整

调整方案

基础数据结构调整方案

预处理调整方案

  • 方案一

加载Dwg文件时进行基础预处理，并存储预处理得到的基础数据。在进行识别时，识别类需要告知识别管理类需要什么数据，识别管理类查看数据仓库中是否存在识别类所需的数据，不存在则根据加工类工厂实例化对应数据加工类,继而对基础数据进行加工成识别类所需数据，储存加工后的数据到数据仓库。作为每次识别的数据源。

  • 方案二

加载Dwg文件时进行基础预处理，并存储预处理得到的基础数据。识别管理类根据调用者输入要识别类型，确定识别所需数据。识别管理类查看数据仓库是否存在数据，不存在则根据识别类提供的成员方法实例化数据加工类初始化其函数指针，然后对基础数据进行加工，储存加工后的数据到数据仓库。得到所需数据，继而进行识别。

基础预处理

主要负责待识别图纸dwg文件实体数据读取，存储到基础数据结构中。

加工处理

新增数据加工模块，各模块在识别过程中对预处理基础数据进行再次加工，进行缓存便于识别模块分析处理。识别模块仅仅能使用加工后的数据

  • 图例识别

    • 读取特征文件
    • 部分块及多段线打散操作
    • 图元分析处理
    • 扫描器构建
    • 提取特征

  • 表格识别相关

    • 按块存储实体及矩阵

  • 文本分析相关

    • 提取对应块表记录中所有文本，进行排序处理

对象构建

  • 预处理类：对数据进行预处理，生成基础数据
  • 预处理数据类：用于存储预处理后的数据，进行管理
  • 数据加工类：根据识别所需数据，进行基础数据加工
  • 加工数据类：加工后的数据结构
  • 加工类工厂类：主要用于实例化加工类
