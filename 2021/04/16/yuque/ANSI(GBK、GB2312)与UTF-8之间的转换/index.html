ANSI(GBK、GB2312)与UTF-8之间的转换

April 16, 2021 编码 本文总阅读量次

为了更清晰地说明 Windows 与 Linux 之间的编码问题， 我决定以代码实践的方式进行展开。

前提说明

首要概念:
**单字节字符串: **即 ACSII 字符串， 单个字节可表示一个字符的字符串。
多字节字符串: 多个字节来表示一个字符， 如 GB18030, UTF-8 等。
**宽字符字符串: **宽字符由操作系统决定宽度。 如 Windows（UTF-16LE）、Linux（UTF-32）。

其次需要明确一个概念: ANSI(GB2312、GBK)与 UTF-8 之间进行转换必须经过 Unicode 作为中转。

编码检查

msvc 编译选项解释:

/source-charset 是用于将程序的源文本解释为在编译之前用于预处理阶段输入的内部表示形式的编码。
/execution-charset 预处理之后，用于编译的代码中的任何字符串或字符文本的内部表示形式。

个人理解是将 source-charset 指定的编码解释源代码用作编译， 而 execution-charset 主要将代码中的字符及字符串按指定编码存储到可执行文件。

详见微软解释/execution-charset与/source-charset

测试结果:

  • 宽字符为 UTF-8
  • 多字节字符为 GB2312

结论:
尽量保持文件编码与/source-charset 编码一致， 否则会造成乱码情况。从以上测试来看，测试check_file_utf8_source_gbk_execution_gbk 源文件格式为 utf-8，而指定/source-charset 为 gbk, 于是编译警告， 但是编译器自动转换代码为 unicode，所以最终字符串都为 utf-8。测试check_file_utf8_source_utf8_execution_gbk同理。而对于测试check_file_gb2312_source_utf8_execution_gbk与测试check_file_gb2312_source_utf8_execution_utf8直接警告非法字符， 导致最终结果未定义。

下面分别对不同编码文件及编译选项的代码进行打印输出:

test_file_utf8_source_utf8_execution_gbk 控制台为 utf-8， 多字节字符串为 GB2312，所以需要转换为 Unicode 然后转换为 UTF-8 进行显示， 而宽字节字符串为 UTF-8，需要指定 local 为 gbk 然后添加一个转换为 utf-8 的转换器才能正常显示。
test_file_utf8_source_utf8_execution_utf8 控制台为 utf-8, 多字节宽字节字符串都为 UTF-8。所以多字节可直接显示如需转换。 而宽字节字符串同上。
test_file_gb2312_source_gbk_execution_gbk 控制台为 gbk, 宽字符为 UTF-8, 多字符为 GB2312。所以多字符串能直接显示， 而宽字符串需要添加 local， 默认转换器为 new std::codecvt_byname<wchar_t, char, std::mbstate_t>(“chs”)，所以无需指定。
test_file_gb2312_source_gbk_execution_utf8 控制台为 gbk, 多字节宽字节字符串都为 UTF-8。多字节字符串需要转换为 Unicode 然后转换为 GBK 进行显示。

编码检查

gcc 编译选项解释:

  • -finput-charset=charset，指定源文件本身的编码方式，默认为 UTF-8(有无 BOM 均可)。例如当我们的源代码文件保存为 GBK 时，则也应当将此选项的值指定为 GBK。
  • -fwide-exec-charset=charset，指定宽字符或宽字符串的字面值常量的内部编码方式，默认为 UTF-32 或 UTF-16，对应 wchar_t 的宽度。wchar_t 的宽度依赖平台实现，windows 实现为 2 字节宽，linux 实现为 4 字节宽。例如指定此选项为 GBK，则宽字符或宽字符串常量将会以 GBK 编码方式存储，而不是默认的 UTF-16 或 UTF-32 编码方式。
  • -fexec-charset=charset，指定窄字符或窄字符串的字面值常量的内部编码方式，默认为 UTF-8。例如指定此选项为 GBK，则窄字符或窄字符串常量将会以 GBK 编码方式存储而不是默认的 UTF-8 编码方式。

由于测试结果一致， 再此就不代码详述了。 代码上存在差异如 codecvt_byname 的参数 Windows 为chs，Linux 为zh_CN.GBK。 system 无法利用 chcp 切换终端编码， 因为 chcp 为 Windows 命令。
