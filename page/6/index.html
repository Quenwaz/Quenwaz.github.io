简单工厂模式

简单工厂模式由输入产品类型，决定生产哪一类产品。通常为枚举值，或特定字符串。

使用场景

当需要生产的产品较少时，这是一种比较合适的方式。客户只需要知道生产产品的类型，无需关心如何生产。因工厂类需要根据类型进行生产，所以导致其内聚性比较低，每当新增产品时，需要在工厂类新增对应生产代码。

字符与基本规则

建议使用括号进行换行，使语句可读性更高。

代码组由不同缩进分隔

核心风格:缩进四个空格宽度，避免使用制表符

推荐使用空格替代制表符的原因：因为不同文本编辑器中的制表符代表的空白宽度不统一，如果你的代码要跨平台，或者不同的编辑器读写，建议不要使用制表符。

标识符

python 标识符规则与大多数计算机语言相似:

  • 第一个字符必须为字母或下划线

专用下划线标识符

  • _xxx: 不用’from module import *’导入
  • _xxx_：系统定义名字
  • __xxx：类中的私有变量名

核心风格:避免用下划线作为变量名的开始，因为下划线对解释器有特殊的意义。
